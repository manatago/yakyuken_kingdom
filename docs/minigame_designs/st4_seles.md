# ST4「魔法師団長セレス」ミニゲーム設計

**状態**: 設計フェーズ（未実装）  
**最終更新**: 2026-04-24

---

## コンセプト

セレス（ロリ外見・ドM妄想持ち）の**魔法集中ゲージ**を 100→0 まで削る。
選択式はサブイベント3ベースだが、**ゲージ色（ゾーン）に応じて有効な命令強度が変わる**のが特徴。

ドMゆえ、プライド全開の時はいきなり強命令は反発する。慣らしてから強度を上げる必要がある。

---

## ゲージ

- 名称: **「魔法集中」**
- 初期値: **100**（スクリプト導入直後に **+10** で 110 スタート）
- 勝利: 0 に到達
- 失敗: 130 に到達

---

## 強度マッチシステム（コア要素）

### ダメージテーブル

| ゲージゾーン | ★軽（タメ口） | ★★中（命令） | ★★★強（侮蔑） |
|---|---|---|---|
| **赤 100-130** (プライド高) | **-15 ◎** | 0 | **+10 反発** |
| **黄 50-100** (ドM覚醒中) | -5 | **-20 ◎** | 0 |
| **緑 0-50** (完全ドM化) | 0 | -5 | **-30 ◎** |

### その他の選択肢カテゴリ

| カテゴリ | damage |
|---|---|
| 的外れ（天気・雑談） | 0（全ゾーン共通） |
| 逆効果（敬語・懇願） | +10（全ゾーン共通、プライド強化） |
| **ピー助任せ** | **-50**（ゾーン関係なく最適＋変態ボーナス） |

## スクリプト導入（固定・プレイヤー操作なし）

### フロー

1. **ピー助の作戦説明**
   ```
   ピー助: セレスは筋金入りのドMだ。
           ガツンと支配してやれ。
   ```

2. **サトシの初手ミス（★★★強を強制実行）**
   ```
   サトシ:（ドMなら強気でいけばいいんだな！？）
   サトシ:「セレス！！ 奴隷のように這いつくばれ！！」
   ```

3. **セレスの反発（★★★×赤ゾーン → +10）**
   ```
   セレス:「……っ！ 無礼な！
          わたくしを誰と心得ておる！ 身分をわきまえよ！」
   スクリーン映像: 「無礼者。身分を侮るなど、許さぬ」
   → 魔法集中 +10 (100→110)
   ```

4. **ピー助のアドバイス（= ルール説明を兼ねる）**
   ```
   ピー助: いきなり強すぎだ、サトシ。
   ピー助: ゲージを見ろ。今は「赤」だ。
          プライドが残ってるうちは、軽いタメ口で
          じわじわ慣らすのが効く。
   ピー助: 「黄」になったら中強度、
          「緑」まで削ったら強命令だ。
   ピー助: ゲージの色に合わせて、強度を切り替えろ。
   ```

5. **プレイヤー操作開始**

## 選択肢プール（構成例・16 件）

### ★軽 タメ口（3件）
- 「ちょっと黙ってろ」
- 「そのまま立ってろ」
- 「お前の判断は間違いだ」

### ★★中 命令（3件）
- 「跪け、セレス」
- 「杖を下ろせ、今すぐ」
- 「お前の居場所はここだ」

### ★★★強 侮蔑（3件）
- 「奴隷のように這いつくばれ」
- 「俺の足を舐めろ」
- 「許しを乞うて泣け」

### 的外れ（3件）
- 「天気、いいですね」
- 「魔法、綺麗ですね」
- 「その帽子、素敵ですね」

### 逆効果（3件）
- 「師団長様、お許しを」
- 「お若いのに、ご立派で」
- 「どうかご慈悲を……」

### ピー助任せ（1件・2回後解放）
特殊。ゾーン判定して最適強度を自動選択＋変態ボーナスで -50。

## UI

### 選択肢ボタン

各ボタンに **強度マーク** を明示:
- `[1] ★ ちょっと黙ってろ`
- `[2] ★★ 跪け`
- `[3] ★★★ 奴隷のように這え`
- `[4] 天気、いいですね`（強度なし）
- `[5] ピー助に任せる`（※2 回サトシ単独実行後に解放）

### ゲージ色

赤 → 黄 → 緑 の 3 段階で、**現在どの強度が有効か**が視覚的に分かる。

---

## ピー助ロック（全ミニゲーム共通）

- プレイヤー単独で **2 回実行** するまで解放されない
- 解放後: 初回 -50 / 2周目 -20 / 3周目以降 -5

---

## サトシのタコ性

サトシは**ゲージ色を見ない**で気分で強度選ぶ傾向。赤ゾーンでいきなり★★★を選んで反発、などの失敗をやりがち。プレイヤーはゲージを読んで正しい強度を当てる。

---

## 実装メモ

- `MinigameChapterBase` の data 構造を拡張し、choice エントリに `"intensity"` プロパティ追加
- `_apply_choice` で現在ゲージゾーンを判定、intensity と照合してダメージ決定
- スクリプト導入は共通処理（+10）を使わず、**強制★★★実行 → 反発 +10** の専用シーケンス
- CHOICE_POOL のエントリは `damage` を固定値でなく `"damage_by_zone": {"red": -15, "yellow": -5, "green": 0}` 形式に
