# サトシ敗北時の共通ナレーション・プール

各サブイベント／本編ステージでサトシがバトルに敗北した際、
個別の脱衣描写を毎回書き起こす代わりに、本ファイルに登録された
パターンからランダムに 1 つ選んで表示する。

## 設計方針

- **目的**: 章ごとの脱衣描写を毎回書き直す手間を削減し、トーンを統一する。
- **位置づけ**: 各章固有の「敗北状況」ナレ (1〜2 行) の **直後** に挿入。
    例: シスター長戦敗北 →
        「番兵に拘束された…」 (章固有)
        + 共通ナレーション (ランダム 1 件)
        → ギルドホーム送還
- **描写レベル**: 「脱がされ始め」までで打ち止め。下着姿は描かない。
    上着のボタンが弾ける／ベルトが緩む／シャツが捲れる
    といった示唆的演出に留め、ピー助が「撤退」を促して打ち切る。
- **ランダム抽出**: 1 セッション中、直前に出たパターンは可能なら
    回避する（実装時に重複回避の責務を持つ）。

## 実装規約

- 各バトルチャプターの `get_lose_redirect()` から呼ばれる
  story_sequence_then_guild_home 系のシーケンスは、章固有のナレ後に
  以下のいずれか 1 件を表示してからギルドホームに戻る。
- 共通ナレーションを表示するための DSL コマンド
  `b.satoshi_lose_narration()` を新設予定。
- パターン定義は実装側 (`BattleChapterBase.SATOSHI_LOSE_NARRATIONS`)
  に const として保持し、本ドキュメントの内容と同期させる。
- パターンを追加・修正したら本ドキュメントとコードの両方を更新する。

## ナレーション・パターン (5 種)

### A. 滑稽 (アップテンポ・ピー助なし)

```
ボタンが次々と弾け飛ぶ。「今度こそ無理だ！」
サトシは半泣きで衣服を抱え、駆け出した。
```

### B. クール風 (短く重い、感情を抑える)

```
サトシは敗北を悟り、捲れた上着を押さえて、静かに身を引いた。
```

### C. 強調 (脱がされ始めの状況描写を強める)

```
シャツが捲れ、ベルトが緩む——なりふり構わず、
サトシは衣服を掻き集めながら、その場を退散した。
```

### D. ピー助介入 (ツッコミ役の典型)

```
ピー助:（小声）
  サトシ、ズボンを抑えろ！ 全力で逃げるぞ！

サトシは情けない悲鳴を上げながら、ピー助に引きずられるようにして
駆け出した。
```

### E. 簡潔 (一言締め)

```
サトシは脱がされかけて、屈辱とともにその場を退散した。
```

## 公開・中継系の例外条項

サブイベント 3 のように「真言の水晶球＋伝声塔」など、サトシの姿が
公衆に中継されうる演出が組み込まれている場合は、サトシ敗北時は
**必ず中継が停止／切断／失敗する** 設定にする。サトシの姿は世間に
晒されない。これは本ドキュメントのナレーション・パターンに優先する
ハードルールとして守る。
